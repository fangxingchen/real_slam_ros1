# 朝向偏移校正指南

## 问题描述
小车到达目标位置后可以转动，但转动的角度和规划的角度一直有一个固定的差距。

## 解决方案
添加了 `YAW_OFFSET` 参数，用于补偿系统误差。

---

## 校正步骤

### 1. 测量偏差角度

1. **设置目标朝向**：
   - 在 RViz 中点击 "2D Nav Goal"
   - 点击目标位置
   - 拖动鼠标设置期望的朝向（记住箭头方向）

2. **观察小车最终朝向**：
   - 小车到达位置并停止旋转后
   - 观察小车箭头方向与期望方向的偏差

3. **计算偏差角度**：
   - 如果小车偏左：偏差为负值
   - 如果小车偏右：偏差为正值
   - 例如：偏左 30° → -0.52 弧度

### 2. 计算弧度值

| 角度（度） | 弧度值 | 角度（度） | 弧度值 |
|-----------|--------|-----------|--------|
| 10° | 0.17 | -10° | -0.17 |
| 20° | 0.35 | -20° | -0.35 |
| 30° | 0.52 | -30° | -0.52 |
| 45° | 0.79 | -45° | -0.79 |
| 60° | 1.05 | -60° | -1.05 |
| 90° | 1.57 | -90° | -1.57 |

**公式**：弧度 = 角度 × π / 180

### 3. 修改参数

编辑 `config/omnidirectional_pid_param.yaml`：

```yaml
# 朝向偏移校正（弧度）
# 正值：顺时针偏移；负值：逆时针偏移
YAW_OFFSET: -0.52  # 例如：补偿 -30° 偏差
```

## 常见偏差原因

### 1. TF 坐标系转换误差
- 不同坐标系之间的朝向定义可能有差异
- 解决：调整 `YAW_OFFSET`

### 2. 路径规划器朝向计算误差
- 全局规划器生成的路径末点朝向可能不准确
- 解决：优先使用 RViz 的 goal 消息（已实现）

### 3. 里程计漂移
- 长时间导航后，里程计可能累积误差
- 解决：使用更精确的定位系统，或定期重定位

### 4. 机械安装误差
- 小车底盘安装时朝向传感器可能有偏差
- 解决：校准 IMU 或编码器安装角度

## 高级调试

### 检查朝向来源优先级

代码按以下优先级获取目标朝向：

1. **RViz Goal**：`/move_base_simple/goal` 消息（鼠标拖拽的朝向）
2. **路径末点**：`/opt_path` 末尾 pose 的朝向
3. **当前朝向**：兜底方案

### 验证朝向计算

```bash
# Python 脚本验证朝向计算
python3 -c "
import math
# 四元数转弧度
def quat_to_yaw(x, y, z, w):
    return math.atan2(2*(w*z + x*y), 1 - 2*(y*y + z*z))

# 示例：四元数 (0, 0, 0.383, 0.924) 对应 45°
yaw = quat_to_yaw(0, 0, 0.383, 0.924)
print(f'Yaw: {yaw:.3f} rad ({yaw*180/math.pi:.1f} deg)')
"
```

## 参数调优建议

| 场景 | 参数调整 |
|------|---------|
| 偏差较大（>20°） | 调整 `YAW_OFFSET` |
| 旋转不稳定 | 降低 `KP_YAW`（如 1.0） |
| 旋转太慢 | 提高 `KP_YAW`（如 2.0） |
| 容差太严 | 提高 `YAW_TOL`（如 0.15） |
| 超时太快 | 提高 `YAW_ADJUST_TIMEOUT`（如 8.0） |
